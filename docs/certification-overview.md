# Certification Overview

## Purpose of this repository

This repository converts the seven preparation-course areas into a practical study system. The emphasis is on choosing and operating Claude responsibly, not merely recognizing definitions.

## Domains covered

| Domain | Core decision |
|---|---|
| Claude Platform & Model Foundations | Which Claude surface and model characteristics fit the task? |
| Prompting & Task Execution | How should the task be specified so it is clear, bounded, and testable? |
| Evaluating & Validating Output | How will quality, grounding, consistency, privacy, and safety be measured? |
| Workflow Integration & Solutions Design | Which steps belong to Claude, deterministic code, tools, or human reviewers? |
| Configuration & Knowledge Management | Which instructions and sources are authoritative, current, and permitted? |
| Governance, Risk & Responsible Use | What controls are required for data, access, high-impact decisions, and incidents? |
| Troubleshooting & Optimization | How can failures be isolated, measured, corrected, and prevented from recurring? |

## Stable concepts versus volatile facts

### Stable concepts

These are appropriate for durable study notes:

- clear success criteria;
- task decomposition;
- evidence grounding;
- structured output contracts;
- representative evaluation sets;
- least privilege;
- human approval for high-impact actions;
- observability and regression testing;
- source authority and freshness; and
- controlled troubleshooting.

### Volatile facts

Verify these in current official sources:

- available model names and aliases;
- context and output limits;
- pricing;
- regional availability;
- feature status;
- API versions;
- product plan entitlements;
- policy text;
- exam registration, format, duration, and passing requirements.

The practice questions intentionally emphasize durable judgment. Product-specific details should be checked before the exam.

## How scenario questions are designed

A scenario normally contains:

1. a business or mission objective;
2. a technical constraint;
3. a risk or failure mode;
4. several plausible actions; and
5. one action that best balances correctness, control, and operational fit.

When two answers appear technically possible, prefer the answer that:

- directly addresses the stated failure;
- introduces the least unnecessary complexity;
- preserves traceability;
- uses deterministic controls where appropriate;
- applies least privilege;
- tests against explicit criteria; and
- keeps qualified humans accountable for high-impact decisions.

## Exam-integrity commitment

This repository does not accept recalled live-exam questions or proprietary course material. The question bank is original and is intended to exercise the same general skills described by the seven study areas, not to predict exact exam content.
