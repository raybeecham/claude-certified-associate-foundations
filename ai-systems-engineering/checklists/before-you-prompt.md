# Before You Prompt

Use this checklist before beginning a substantial task.

## Problem

- [ ] I can state the business problem without naming a model or prompt.
- [ ] I know who will use the output.
- [ ] I know what decision, deliverable, or action the output supports.
- [ ] I have defined measurable success criteria.
- [ ] I understand the consequence of a wrong answer.

## Workspace

- [ ] I decided whether the task is one-off or recurring.
- [ ] I selected Chat, Project, Research, Artifact, an application, or a justified combination.
- [ ] I am not creating persistence or integration that the task does not require.

## Capabilities

- [ ] Stable context is separated from current task input.
- [ ] A repeated procedure is documented rather than retyped.
- [ ] Calculations and transformations will use code or deterministic logic.
- [ ] Current facts will come from current authorized sources.
- [ ] Memory is used only for appropriate continuity.
- [ ] Tools and connectors have narrow permissions.

## Model

- [ ] I selected a starting model based on structure, ambiguity, volume, latency, and consequence.
- [ ] I can explain what the model choice buys and what it costs.
- [ ] I know what evidence would justify a lower or higher tier.

## Context

- [ ] Only relevant and authorized information is included.
- [ ] Sources are labeled by authority, version, and date.
- [ ] Conflicting sources have a resolution rule.
- [ ] I know what belongs in the current thread, Project, Memory, or external record.
- [ ] I have a checkpoint or restart plan for long work.

## Prompt

- [ ] The objective is explicit.
- [ ] The audience and use are explicit.
- [ ] Instructions and source material are clearly separated.
- [ ] The required process is defined.
- [ ] The output format is testable.
- [ ] Missing-data and uncertainty behavior are defined.
- [ ] The model is permitted to say `unknown` or escalate.

## Validation

- [ ] I know how factual claims will be checked.
- [ ] I know how calculations will be reconciled.
- [ ] I know who reviews the output.
- [ ] I know what failure blocks use or publication.
- [ ] Human responsibility remains explicit.

## Final gate

Proceed only when the prompt is the next engineering step, not the first guess.
