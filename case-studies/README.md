# Case Studies

These case studies integrate multiple domains. They are designed for written analysis, architecture diagrams, and peer review.

## Recommended method

For each case:

1. Identify users, mission outcome, data, sources, tools, and side effects.
2. Define measurable success criteria and hard release gates.
3. Select the simplest suitable interaction pattern.
4. Partition Claude, code, tools, storage, and human responsibilities.
5. Threat-model untrusted content and overly broad access.
6. Design evaluation, monitoring, rollback, and incident handling.
7. Write an architecture decision record.
8. Review the solution against all seven modules.

Do not use real organizational secrets or controlled data in public artifacts.
