# Module 6: Governance, Risk & Responsible Use

## Why this domain matters

Responsible use is not a final review checkbox. Data handling, human accountability, least privilege, policy, monitoring, change management, and incident response must be designed into the system before deployment.

## Learning objectives

By the end of this module, you should be able to:

- classify data and verify an approved processing environment;
- identify high-impact uses that require meaningful human oversight and disclosure;
- threat-model prompt injection, data exfiltration, unauthorized action, and unsafe dependencies;
- design least-privilege access and minimized logging;
- define change review and exception authority;
- recognize prohibited or restricted use;
- create monitoring and incident response paths; and
- distinguish model assistance from delegated accountability.

## Exam lens

A nominal “human in the loop” is insufficient when the reviewer lacks expertise, evidence, authority, or time. The best answer preserves human accountability before a consequential action and applies current policy and organizational controls.

## Completion criteria

- [ ] I can classify data before selecting an environment.
- [ ] I can define meaningful human review.
- [ ] I can identify prompt-injection controls.
- [ ] I can outline containment and incident response.
- [ ] I completed the threat-model lab and scored at least 80% on the quiz.
