# Study Modules

Each module uses the same structure:

- `README.md`: purpose, objectives, completion criteria, and official reading
- `notes.md`: exam-focused concepts and decision patterns
- `lab.md`: applied exercise with acceptance criteria and a solution outline
- `flashcards.md`: twelve active-recall cards
- `quiz.md`: eight original scenario questions with rationales

## Study sequence

The modules build on one another. Platform choices affect prompting. Prompt contracts enable evaluation. Evaluation informs workflow design. Configuration and knowledge govern what the workflow can trust. Governance defines acceptable use. Troubleshooting uses evidence from every layer.

Study them in order on the first pass. Later review should follow weak areas.
